---
description: Review current changes, commit them with a descriptive message, and push to GitHub
---

1. Run `git status` to see changed files.
2. Run `git diff` to review specific changes.
3. Analyze the changes and generate a concise, descriptive commit message.
4. Stage changes using `git add .` (or specific files if appropriate).
5. Commit changes using `git commit -m "<GENERATED_MESSAGE>"`, replacing <GENERATED_MESSAGE> with the message you generated.
6. Push changes using `git push`.
